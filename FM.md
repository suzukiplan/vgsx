# 設計メモ: FM音源エミュレーション

VGS-X は YM2612 (OPN2) エミュレータを搭載して、FM音源の楽曲を再生することができます。

FM音源エミュレータは [ymfm](./LICENSE-ymfm.txt) をベースに次のような改造をしています:

- `ym2612` / `ym3438` / `ymf276` の出力挙動を選択できるようにしました。
- YM2612 の出力近似を、従来の「6チャンネルを単純に合算して平均する方式」から、ピン出力に着想を得たモデルへ変更しました。このモデルでは、アクティブ出力フェーズと side-off 時の sign 出力を分けて扱います。
- `ch_out` / `output` の2段ラッチ状態を追加し、YM2612 の平均出力が同一サイクルのFM合算値ではなく、ラッチ済みのチャンネル値に基づくようにしました。
- `ymf276` では高解像度の mixed-DAC パスを維持し、`ym3438` には YM2612 固有の DAC 不連続モデルを適用しないようにしました。

これらは、Nuked OPN2 や YMF276-LLE などの OPN2 エミュレータの実装内容からヒントを得て実装した特徴です。

> _なお、Nuked OPN2 と YMF276-LLE は GPL ライセンスのフリーソフトウェアです。そのため、VGS-X ではそれらのソースコードを流用せず、公開されている挙動や考え方を参考にしながら、独自に実装しています。_

## アナログ・エミュレーション

FM音源の音が生成されるメカニズム:

1. プログラムが FM音源IC のレジスタを更新
2. FM音源IC が デジタル波形 を生成
3. DAC が デジタル波形 を アナログ波形 に変換
4. オペアンプ などのフィルタが アナログ波形 を加工
5. 加工された アナログ波形 を LINE OUT へ出力
6. スピーカーなど が LINE OUT から入力された音を再生


一般的なFM音源エミュレータは FM音源IC のエミュレーションを目的としています。

具体的には上記の `1` と `2` がエミュレータの責務パートです。

しかし、VGS-X ではその後段にあたる `3` 〜 `4` の要素も簡易的にモデル化し、IC単体の出力だけでなく、DAC後段の音質変化も含めた再生を目指しています。

## 比較対象

以下4つの機器・エミュレータで同一の曲を再生して波形特性を比較しました:

1. YM2612 実チップ環境: Project RE:birth RE1-YM2612/3438
2. Nuked OPN2 (別の OPN2 エミュレータ)
3. VGS-X のアナログエミュレーション無し: `clean` プリセット
4. VGS-X のアナログエミュレーション有り: `re1e` プリセット

## 比較結果

比較には `tools/vgm2wav/test.vgm` を使用しました。
実機録音 `test_re1.wav` は録音先頭に待ち時間があるため、RMS包絡の立ち上がりから `3.43s` を曲頭として頭出ししました。
`clean` / `nuked` / `re1e` はレンダリング結果の先頭を曲頭として扱っています。

評価値は、各区間をRMS正規化したうえで、以下の11帯域の平均スペクトル形状を比較したものです。

- `20-80Hz`
- `80-160Hz`
- `160-315Hz`
- `315-630Hz`
- `630Hz-1.25kHz`
- `1.25-2.5kHz`
- `2.5-5kHz`
- `5-8kHz`
- `8-12kHz`
- `12-16kHz`
- `16-22kHz`

数値は `re1` を基準にした帯域プロファイルのRMSEです。
小さいほど `re1` に近いスペクトルバランスです。

| 比較区間 | clean | Nuked OPN2 | re1e |
|---|---:|---:|---:|
| 0-2s 前打ち中心 | 5.797 dB | **5.714 dB** | 5.914 dB |
| 0-4s | **1.554 dB** | 1.702 dB | 1.556 dB |
| 0.5-6.5s メロディ中心 | 1.059377 dB | 1.321459 dB | **1.059376 dB** |
| 0-8s 全体 | 1.004 dB | 1.154 dB | **1.000 dB** |

0.5-6.5s のメロディ中心区間では、`re1e` は `clean` とほぼ同等ですが、RMSEではわずかに `clean` を下回りました。
0-8s 全体でも `re1e` が最小です。
一方、前打ち中心の0-2sでは Nuked OPN2 が最も近く、`re1e` は `clean` よりわずかに遠くなっています。

0-8s 全体での `re1e` の改善率は以下です。

| 比較元 | RMSE | re1eとの差 | 改善率 |
|---|---:|---:|---:|
| clean | 1.004 dB | -0.004 dB | 0.4% |
| Nuked OPN2 | 1.154 dB | -0.154 dB | 13.4% |

`re1e` は `clean` に対して非常に薄い補正です。
0-8s 全体での `re1e - clean` の帯域差は以下です。

| 帯域 | re1e - clean |
|---|---:|
| 20-80Hz | +0.006 dB |
| 80-160Hz | +0.005 dB |
| 160-315Hz | +0.005 dB |
| 315-630Hz | +0.005 dB |
| 630Hz-1.25kHz | +0.005 dB |
| 1.25-2.5kHz | +0.006 dB |
| 2.5-5kHz | +0.004 dB |
| 5-8kHz | +0.001 dB |
| 8-12kHz | -0.005 dB |
| 12-16kHz | -0.012 dB |
| 16-22kHz | -0.019 dB |

つまり、`re1e` は低域を積極的に補強するプリセットではなく、`clean` のスペクトルをほぼ維持しながら、最上位帯域だけをわずかに丸める設定です。

## 考察

今回の比較では、`clean` の時点で実機録音 `re1` にかなり近いスペクトルバランスになっていました。
そのため、`re1e` で大きなアナログ補正を加えると、特にメロディ中心区間では低域過多または高域不足になり、かえって `re1` から遠ざかります。

`re1e` の最終的な調整では、低域補強、ノッチ、非線形歪み、バスサチュレーションを使わず、ごく弱いローパスだけを残しました。
これにより、0-8s 全体とメロディ中心区間では `clean` よりわずかに `re1` へ近づきました。
ただし改善幅は小さく、`clean` と `re1e` は聴感上もかなり近い関係です。

Nuked OPN2 は前打ち中心の0-2sでは最も `re1` に近い結果でした。
一方で、メロディ中心区間と0-8s全体では `clean` / `re1e` よりRMSEが大きく、曲全体の平均的なスペクトルバランスでは VGS-X のほうが `re1` に近い傾向です。

以上から、`re1e` は「実機の録音経路まで強く再現する」プリセットではなく、`clean` を基準に Project RE:birth RE1-YM2612/3438 の録音へごく薄く寄せるためのプリセットと位置づけるのが妥当です。
実機FM音源の軽さを補うための音作りを目的にする場合は、`re1e` よりも `real` のような別プリセットで扱うほうが、意図と測定結果の整合が取りやすくなります。


## `real` プリセット（デフォルト）

`re1e` は、Project RE:birth RE1-YM2612/3438 の録音特性へごく薄く寄せることを目的としたプリセットです。
一方、VGS-X のデフォルトである `real` は、実機録音への一致だけを目的としたものではなく、ゲーム内で使いやすいFM音源としての聴感も重視しています。

実機FM音源の録音では、環境によって低域が弱くなり、曲全体がやや軽く聞こえる場合があります。
そのため、まず低域を少し補強した `subtle` プリセットを作成しました。
ただし、`subtle` は低域補強の副作用として、高域やアタック感がわずかに弱くなる傾向がありました。

そこで、低域の厚みをある程度確保しつつ、高域やアタック感も保つように調整した `real` プリセットを作成し、VGS-X のデフォルトとしました。
`real` は厳密な実機録音再現ではなく、実機FM音源らしさとゲーム用途での聴きやすさを両立するための実用プリセットです。
